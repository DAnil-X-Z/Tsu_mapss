2-й модуль: Алгоритмы приближенных вычислений
Проект для студентов Томского государственного университета (ТГУ)
Формат: групповая работа (1–4 человека)
Цель: реализовать набор алгоритмов приближённых вычислений + их визуализацию + полноценный интерфейс в виде мобильного приложения.

Важно:

Запрещено использовать готовые библиотеки/решения алгоритмов и генеративные ИИ для генерации кода.
Обучающие данные/датасеты можно брать готовые.
Всё, что касается реализации алгоритмов и их работы — только руками команды.
Если делаете бонусный вариант — базовый можно пропустить (если не указано обратное).
Что нужно сделать: пошаговый план
Этап 1: Подготовка ресурсов — разметка карты
Обязательно для всех алгоритмов!

Выбрать карту:
Скриншот/экспорт из 2GIS, Google Maps или Яндекс.Карт (роща ТГУ + прилегающие территории).
Разметить карту на квадраты (grid):
Рекомендуемый размер ячейки: 5–10 метров (подбирайте так, чтобы дорожки были проходимы).
Формат хранения: 2D-массив (List<List> или Grid в Flutter).
0 — проходимо (дорожки, газоны без препятствий)
1 — препятствие (здания, внутренние помещения, газоны, заборы)
Создать файл data/map_grid.json или .dart с готовой разметкой.
Добавить возможность редактирования сетки в приложении (для бонусов).
Проверить:
Все реальные дорожки должны быть проходимы (протестировать A* вручную).
Сохранить координаты ключевых точек:
кафе/столовые (для генетического и кластеризации)
достопримечательности рощи
коворкинги/библиотеки/аудитории
главные входы зданий
Добавить GPS-координаты (lat/lng) для каждой ячейки — понадобится для реального положения пользователя.
Результат этапа: готовый файл карты + визуализация сетки в приложении (можно на отдельном экране "Debug Map").

Этап 2: Структура мобильного приложения
Приложение должно иметь 3 главных раздела (как указано в задании):

Навигация по кампусу (A*)
Общепит (Кластеризация + Генетический + Дерево решений + Нейронная сеть)
Коворкинг и экскурсия (Муравьиный алгоритм)
Экраны (минимальный набор):

Splash / Onboarding
Главный экран (BottomNavigationBar или Drawer) с 3 вкладками
Карта (общая для всех алгоритмов)
Экран выбора алгоритма / функции
Экран визуализации каждого алгоритма (анимация!)
Экран настроек + профиль студента
Дополнительно для доп. баллов:

Определение текущего положения (Geolocator)
Темная/светлая тема в стиле ТГУ
Сохранение истории маршрутов
Этап 3: Реализация алгоритмов
3.1 A* (строим кратчайшие маршруты до кафе)
Базовый вариант:

На карте выбрать стартовую и конечную точку (тапом или поиском).
Запустить A* с одной эвристикой (Manhattan или Euclidean).
Показать маршрут (полилиния) или сообщение «маршрута нет».
Вывести длину пути и время (скорость 5 км/ч).
Бонусный вариант:

Возможность расставлять временные препятствия (ограждающие ленты — тап + drag).
Анимация поиска в реальном времени:
открытые клетки (рассматриваемые)
закрытые клетки
путь, который строится на каждой итерации
Доп. баллы: определение текущего положения пользователя (Geolocator) как стартовой точки.
Визуализация: цвета (зелёный — путь, красный — препятствия, жёлтый — открытые, синий — закрытые).

3.2 Алгоритм кластеризации (определяем зоны еды)
Базовый вариант:

На карте расставить точки кафе (или загрузить готовые координаты).
Реализовать K-means (или DBSCAN — на выбор).
Показать кластеры цветом + границы областей.
Возможность менять количество кластеров (k).
Бонусный вариант:

Реализовать 2–3 метрики расстояния:
Евклидово
Пешеходное (через A* — самое точное!)
Показывать, какие точки «спорные» (меняют кластер в зависимости от метрики) + легенду.
3.3 Генетический алгоритм (маршрут покупки еды — TSP с условиями)
Базовый вариант:

Создать список блюд и заведений (минимум 8–10 заведений с меню).
Пользователь выбирает нужные блюда (чекбоксы).
Генетический алгоритм:
Популяция 50–100 особей
Кроссовер, мутация, элитизм
Фитнес: время + штрафы за отсутствие блюда/посуды/время работы
Показывать лучшую особь на каждой итерации (анимация).
Учитывать:
расстояния (A*)
время работы заведений
скорость 5 км/ч
ограничения меню
Результат: готовый маршрут + общее время.

3.4 Муравьиный алгоритм (TSP + оптимизация коворкинга)
Базовый вариант:

Пользователь выбирает достопримечательности (чекбокс-лист).
Муравьиный алгоритм (ACO) строит путь обхода всех выбранных точек.
Показывать маршрут и длину.
Бонусный вариант (Оптимизация коворкинга):

Ввод количества студентов.
Начальная точка — группа студентов.
Локации: библиотека, читалки, коворкинги (с вместимостью и комфортом).
Муравьи исследуют пространство, оставляют феромоны.
Учитывать: длина пути + комфорт + уже выбранные муравьи.
Анимация муравьёв (рекомендуется видео: https://youtu.be/emRXBr5JvoY).
3.5 Дерево решений (выбираем, куда пойти на обед)
Базовый вариант:

Создать экран загрузки CSV-текста (или встроить пример из задания).
Реализовать дерево решений (ID3/C4.5 — вручную).
Визуализировать дерево (можно в виде списка узлов или простого графа).
Ввод новых данных → проход по дереву → рекомендация + путь по узлам.
Бонус: оптимизация размера дерева (pruning) для красивой визуализации на телефоне.

Данные: использовать CSV из задания + можно расширить признаки.

3.6 Нейронная сеть (оценка заведения)
Базовый вариант:

Экран рисования 5×5 пикселей (палец — чёрный/белый).
Простая нейронная сеть (MLP или даже Perceptron) на 25 входов.
Обучить на 10–20 примерах цифр 0–9.
Показывать распознанную цифру (оценка 0–9).
Бонусный вариант:

Сетка 50×50.
Более мощная сеть (можно CNN, но вручную — без библиотек).
Сохранение нарисованных оценок в историю.

Реализовать экран «Помощь с навигацией», «Обед», «Коворкинг».
Добавить Geolocation (текущая позиция).

на реальном устройстве

скриншоты
инструкция по запуску
